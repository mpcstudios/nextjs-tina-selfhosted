 MPC STUDIOS — NEXT.JS + TINACMS SELF-HOSTED STARTER
=====================================================
This is the MPC Studios starter template for building client websites
with self-hosted TinaCMS, Next.js 15, Tailwind CSS, and Claude Code.

There are three setup paths — pick the one that fits:

  FASTEST: One-click Vercel deploy (see README.md Deploy button)
  OPTION A: Human step-by-step
  OPTION B: Claude Code does the work


================================================================
WHAT'S IN THIS TEMPLATE
================================================================

CLAUDE.md
  The main instruction file for Claude Code. Tells Claude the
  stack, folder structure, code conventions, and TinaCMS patterns.

PROJECT_BRIEF.md
  Where you describe the client project. Fill this out completely
  before opening Claude Code.

AGENTS.md
  Auto-generated by Next.js. Do not edit this file.

.env.example
  All required environment variables with comments explaining
  where to get each one.

tina/
  TinaCMS configuration, database setup, and content collections.
  - tina/config.tsx — main CMS config (auth, media, collections)
  - tina/database.ts — database layer (local dev or Upstash Redis)
  - tina/collections/ — content type definitions (page, post)

pages/api/
  API routes for TinaCMS backend auth and S3 media uploads.
  These use the Pages Router (required by TinaCMS).

content/
  Markdown and JSON content files. Git-backed — edits from the
  CMS are committed to the repo automatically.

patches/
  pnpm patches for ESM/CJS compatibility. Do not delete.

/agents folder
  Claude Code skills for design, SEO, QA, and image generation.


================================================================
PREREQUISITES
================================================================

You need these installed before starting:

  - Node.js 22+
  - pnpm (install: npm install -g pnpm)
  - GitHub CLI (install: https://cli.github.com)
  - Claude Code (install: npm install -g @anthropic-ai/claude-code)

For Vercel deployment you will also need:

  - A Vercel account linked to the mpcstudios GitHub org
  - An AWS S3 bucket + IAM credentials (for media uploads)
  - A GitHub Personal Access Token with repo scope


================================================================
FASTEST — ONE-CLICK VERCEL DEPLOY
================================================================

Click the "Deploy with Vercel" button in README.md (or open
this URL in your browser):

  https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fmpcstudios%2Fnextjs-tina-selfhosted&env=GITHUB_PERSONAL_ACCESS_TOKEN,NEXTAUTH_SECRET,S3_ACCESS_KEY,S3_SECRET_KEY,S3_REGION,S3_BUCKET,ENABLE_EXPERIMENTAL_COREPACK&envDescription=See%20.env.example%20in%20the%20repo%20for%20details%20on%20each%20variable.%20ENABLE_EXPERIMENTAL_COREPACK%20should%20be%20set%20to%201.&envLink=https%3A%2F%2Fgithub.com%2Fmpcstudios%2Fnextjs-tina-selfhosted%2Fblob%2Fmain%2F.env.example&project-name=my-tina-site&stores=%5B%7B%22type%22%3A%22kv%22%7D%5D

This automatically:
  1. Copies the repo to your GitHub account
  2. Creates a Vercel project
  3. Creates an Upstash KV database (sets KV env vars for you)
  4. Prompts you for the remaining env vars

Have these ready before clicking:
  - GITHUB_PERSONAL_ACCESS_TOKEN (GitHub → Settings → Developer
    Settings → Personal Access Tokens → repo scope)
  - NEXTAUTH_SECRET (run: openssl rand -base64 32)
  - S3_ACCESS_KEY, S3_SECRET_KEY, S3_REGION, S3_BUCKET (from AWS)
  - ENABLE_EXPERIMENTAL_COREPACK = 1

After deploy (~3 minutes):
  1. Visit https://your-site.vercel.app/admin
  2. Log in with: tinauser / tinarocks
  3. Clone the repo locally: git clone [your-repo-url]
  4. Run: pnpm install && pnpm dev
  5. Fill out PROJECT_BRIEF.md
  6. Open Claude Code and start building


================================================================
OPTION A — HUMAN SETUP (step by step)
================================================================

STEP 1: CREATE A NEW REPO FROM THE TEMPLATE

  1. Go to github.com/mpcstudios/nextjs-tina-selfhosted
  2. Click "Use this template" → "Create a new repository"
  3. Name it after the client (e.g., "client-name")
  4. Clone it:
       git clone git@github.com:mpcstudios/client-name.git
       cd client-name
  5. Install dependencies:
       pnpm install


STEP 2: FILL OUT PROJECT_BRIEF.md

  Open PROJECT_BRIEF.md and fill out every section:
  - The Basics (client name, business, offerings)
  - The Goal (what the site should accomplish)
  - The Audience (who visits, what they need)
  - Brand & Tone (visual direction, reference sites, colors)
  - Pages & Content (every page and its sections)
  - Competitors (2-3 URLs, how to stand out)


STEP 3: TEST LOCALLY

  Start the dev server:
    pnpm dev

  - Site loads at http://localhost:3000
  - CMS admin at http://localhost:3000/admin
  - Click "Enter Edit Mode" (no login needed in dev)
  - Edit content visually — changes save to local files


STEP 4: BUILD THE SITE

  Open Claude Code in the project folder:
    claude

  Tell it what to build:
    "Read PROJECT_BRIEF.md and build the homepage."
    "Add a blog listing page that pulls from the post collection."
    "Create a new TinaCMS collection for team members."


STEP 5: SET UP VERCEL

  a) Import the repo on vercel.com → Add New Project

  b) Add Upstash KV:
     Vercel dashboard → Storage → Create Database → KV
     This auto-sets KV_REST_API_URL and KV_REST_API_TOKEN.

  c) Set environment variables (Settings → Environment Variables):

     NEXTAUTH_SECRET          → run: openssl rand -base64 32
     GITHUB_PERSONAL_ACCESS_TOKEN → GitHub → Settings → Developer Settings
                                     → Personal Access Tokens → repo scope
     S3_ACCESS_KEY            → from AWS IAM
     S3_SECRET_KEY            → from AWS IAM
     S3_REGION                → e.g., us-east-1
     S3_BUCKET                → your S3 bucket name
     ENABLE_EXPERIMENTAL_COREPACK → 1

  d) Deploy — push to main, Vercel auto-deploys


STEP 6: LOG IN TO THE CMS

  Go to https://your-site.vercel.app/admin
  Default login: tinauser / tinarocks
  You will be prompted to change the password on first login.


================================================================
OPTION B — CLAUDE CODE SETUP (let Claude do the work)
================================================================

This is for coworkers who prefer to have Claude Code handle
the setup. You need:

  Required:
    - GitHub CLI authenticated (run: gh auth login)
    - pnpm installed

  Optional but recommended:
    - Vercel CLI authenticated (run: vercel login)
      If installed, Claude Code can set env vars, link projects,
      and check deployments without touching the Vercel dashboard.

Open a terminal and start Claude Code:
  claude

Then give it these instructions — one at a time, waiting for
each to finish before giving the next:


STEP 1: CREATE THE REPO

  "Create a new repo called mpcstudios/CLIENT-NAME using
  the template mpcstudios/nextjs-tina-selfhosted. Clone it
  to the current directory and run pnpm install."

  Claude Code will run:
    gh repo create mpcstudios/CLIENT-NAME --template mpcstudios/nextjs-tina-selfhosted --public
    git clone git@github.com:mpcstudios/CLIENT-NAME.git
    cd CLIENT-NAME && pnpm install


STEP 2: FILL OUT THE BRIEF

  "Open PROJECT_BRIEF.md and help me fill it out. The client
  is [describe the client]."

  Claude Code will walk you through each section.


STEP 3: TEST LOCALLY

  "Run pnpm dev and verify the site loads."

  Claude Code will start the dev server. Open http://localhost:3000
  in your browser. CMS admin is at /admin.


STEP 4: BUILD THE SITE

  "Read PROJECT_BRIEF.md and build the homepage. Start with
  the hero section and overall page structure."

  Keep giving Claude Code instructions for each page/feature.


STEP 5: SET UP VERCEL

  If you have the Vercel CLI installed (recommended):

  "Link this project to Vercel under the mpcstudios team,
  then set up all the environment variables. Here are my
  credentials:
    - S3_ACCESS_KEY: [your key]
    - S3_SECRET_KEY: [your secret]
    - S3_REGION: [your region]
    - S3_BUCKET: [your bucket]
    - GITHUB_PERSONAL_ACCESS_TOKEN: [your token]
  Generate a NEXTAUTH_SECRET for me. Also set
  ENABLE_EXPERIMENTAL_COREPACK to 1.
  Then create an Upstash KV database on the Vercel dashboard
  and tell me when it's done."

  Claude Code will run:
    vercel link --scope mpcstudios
    openssl rand -base64 32  (generates NEXTAUTH_SECRET)
    vercel env add NEXTAUTH_SECRET production
    vercel env add GITHUB_PERSONAL_ACCESS_TOKEN production
    vercel env add S3_ACCESS_KEY production
    vercel env add S3_SECRET_KEY production
    vercel env add S3_REGION production
    vercel env add S3_BUCKET production
    vercel env add ENABLE_EXPERIMENTAL_COREPACK production

  NOTE: The Upstash KV database must still be created in the
  Vercel dashboard (Storage → Create Database → KV). This
  auto-sets KV_REST_API_URL and KV_REST_API_TOKEN. There is
  no CLI command for this yet.

  If you do NOT have the Vercel CLI:

  a) Go to vercel.com → Add New Project → import the repo
  b) Vercel dashboard → Storage → Create KV Database
  c) Tell Claude Code:
     "Generate a NEXTAUTH_SECRET for me."
     Then copy the output and all other env vars into
     Vercel → Settings → Environment Variables manually.


STEP 6: DEPLOY AND VERIFY

  "Commit all changes, push to GitHub, and check the Vercel
  deployment status."

  Claude Code will run:
    git add -A && git commit -m "Initial site build" && git push
    vercel ls  (if Vercel CLI is installed)

  Once deployed (~3 minutes), visit:
    https://your-site.vercel.app/admin

  Log in with: tinauser / tinarocks

  If there are any issues, tell Claude Code:
  "Check the Vercel production logs for errors."

  Claude Code will run:
    vercel logs --environment production --expand


================================================================
SKILLS & AGENTS
================================================================

WEBSITE DESIGN SKILL (agents/website-design-skill.md)
  Automatically applied to all design work. Covers layout,
  typography, color, interaction, accessibility, and copy.

SEO SKILL (agents/seo-skill.md)
  Automatically applied to all pages. Covers metadata, Open
  Graph tags, structured data, sitemap, and robots.txt.

QA SKILL (agents/qa-skill.md)
  Run when the site is ready for review:
    "Run QA on this site"

IMAGE GENERATION SKILL (agents/image-gen-agent.md)
  Optional. Requires a Gemini API key:
    "Use the image gen agent to generate images for this project"


================================================================
TINACMS QUICK REFERENCE
================================================================

ADMIN PANEL
  /admin — visual content editor with live preview

LOCAL DEV
  pnpm dev — local mode, no auth, edits save to filesystem
  pnpm dev:prod — production mode, requires .env.local with
                   Redis + GitHub credentials

CONTENT FILES
  content/pages/ — page content (markdown)
  content/posts/ — blog posts (markdown)
  content/users/ — CMS user accounts (JSON)

ADDING A NEW COLLECTION
  1. Create tina/collections/your-collection.ts
  2. Import and add to collections array in tina/config.tsx
  3. Create content/your-collection/ directory
  4. Add a sample markdown file
  5. Run pnpm dev to generate types

DEFAULT CMS USER
  Username: tinauser
  Password: tinarocks
  (password change required on first login)


================================================================
NEED HELP?
================================================================

  Claude Code confused?  → Close it, reopen, try again
  Vercel build failing?  → Check Vercel build log, paste error
                           into Claude Code
  TinaCMS admin broken?  → Check Vercel runtime logs:
                           vercel logs --environment production
  Auth not working?      → Verify NEXTAUTH_SECRET and KV env vars
                           are set in Vercel
  Git issues?            → Paste the error into Claude Code
