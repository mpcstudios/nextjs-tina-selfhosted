 MPC STUDIOS — NEXT.JS STARTER
==============================
This is the MPC Studios starter template for building client websites using
Next.js, Tailwind CSS, and Claude Code. Follow this guide every time you
start a new project.

If anything is unclear, refer to the NEED HELP section at the bottom of this file.


================================================================
WHAT'S IN THIS TEMPLATE
================================================================

CLAUDE.md
  The main instruction file for Claude Code. It tells Claude what
  stack we use, how we organize code, and which skills to follow.
  You only need to read the top section — everything else is
  already set up for you.

PROJECT_BRIEF.md
  Where you describe the client project. Fill this out completely
  before opening Claude Code. The better you fill this out, the
  better Claude Code performs.

AGENTS.md
  Auto-generated by Next.js. Tells Claude Code how to write
  modern Next.js code. Do not edit this file.

.env.example
  A list of environment variables the project needs. You will
  copy this to .env.local and fill in the real values.

vercel.json
  Deployment config and security settings for Vercel.
  Do not edit this file.

.prettierrc
  Code formatting rules. Do not edit this file.

/agents folder
  Contains skills that Claude Code can follow for specific tasks.
  See the SKILLS section below for how to use them.


================================================================
STEP 1 — START A NEW PROJECT FROM THE TEMPLATE
================================================================

1. Go to github.com/mpcstudios/nextjs-starter

2. Click the green "Use this template" button
   → Click "Create a new repository"

3. Name the repo after the client (e.g. "frontera-materials")
   → Leave it set to Private
   → Click "Create repository"

4. Open Terminal on your Mac

5. Navigate to where you keep your projects:
      cd ~/Documents

6. Clone the new repo to your computer:
      git clone https://github.com/mpcstudios/YOUR-REPO-NAME.git

7. Move into the project folder:
      cd YOUR-REPO-NAME

8. Install dependencies:
      npm install

9. Open the project in your editor:

   VS Code:
      code .
   (If "code ." doesn't work, open VS Code manually and go to
   File → Open Folder → select your project folder)

   Claude Code only (no separate editor needed):
      claude
   (Claude Code can read and edit files directly — you do not
   need VS Code if you are working entirely in Claude Code)


================================================================
STEP 2 — FILL OUT PROJECT_BRIEF.md
================================================================

This is the most important step. Open PROJECT_BRIEF.md in your
editor or ask Claude Code to open it:
   "Open PROJECT_BRIEF.md and I will fill it out"

Fill out every section before you do anything else.

THE BASICS
  Client name, site type (marketing, e-commerce, etc.), what the
  business does, what they offer, what sets them apart, their
  location or service area, and the live URL if you know it.

THE GOAL
  What should this site accomplish? What is the one action you
  want visitors to take? What does success look like?

THE AUDIENCE
  Who is visiting this site? What do they need to feel and know?

BRAND & TONE
  This section has the biggest impact on design quality.
  - Describe the tone and personality
  - Describe the visual direction (dark/light, minimal/rich, etc.)
  - Add 2-3 reference website URLs with notes on what you like
  - List any existing brand colors, fonts, or assets

PAGES & CONTENT
  List every page and the sections it needs. Be specific.
  Example:
    Home: hero, services overview, about snapshot, CTA banner
    About: company story, team section, values
    Contact: form, phone number, address, Google Maps

COMPETITORS
  Add 2-3 competitor URLs. Note how this site should stand out.

The more detail you put in here, the less back-and-forth you will
have with Claude Code later.


================================================================
STEP 3 — CLAUDE.md IS ALREADY SET UP
================================================================

You do not need to edit CLAUDE.md. It automatically references
PROJECT_BRIEF.md for project context and points to the agent
skills for design, SEO, and QA.

The only time you would edit CLAUDE.md is if you add a new
integration (e.g. Stripe, Sanity) that Claude Code needs to
know about. Add it to the Key Integrations line at the top.


================================================================
STEP 4 — SET UP ENVIRONMENT VARIABLES
================================================================

1. In your terminal, copy the example file:
      cp .env.example .env.local

2. Open .env.local in your editor or ask Claude Code:
      "Open .env.local so I can add the environment variables"

3. Fill in the real values for any variables the project needs
   (API keys, database URLs, etc.)

IMPORTANT: Never commit .env.local to GitHub. It is already
in .gitignore so this should happen automatically — but double
check before pushing if you have added sensitive keys.


================================================================
STEP 5 — OPEN CLAUDE CODE AND START BUILDING
================================================================

Claude Code is an AI coding assistant that lives in your terminal.
It reads CLAUDE.md, PROJECT_BRIEF.md, and the agent skills
automatically when you open a project.

OPENING CLAUDE CODE
  In your terminal, make sure you are in your project folder:
      cd ~/Documents/YOUR-REPO-NAME

  Then open Claude Code:
      claude

HOW TO TALK TO CLAUDE CODE
  Be specific about what you want. Good examples:

  "Read PROJECT_BRIEF.md and build the homepage. Start with
  the hero section and overall page structure."

  "Build the About page based on the project brief."

  "Add a contact form to the contact page that sends email
  via Resend."

  "Fix the mobile navigation — it breaks on small screens."

TIPS
  - Always let Claude Code finish a task before giving it the next one
  - If the output isn't right, describe what's wrong specifically
  - You can ask Claude Code to explain what it built if you're unsure
  - Claude Code remembers the context of your project within a session
    but starts fresh each time you open it — that's why PROJECT_BRIEF.md
    and CLAUDE.md are so important


================================================================
SKILLS & AGENTS
================================================================

Skills are instruction files that tell Claude Code how to handle
specific tasks. They live in the /agents folder.

WEBSITE DESIGN SKILL (agents/website-design-skill.md)
  Automatically applied to all design work. Covers layout,
  typography, color, interaction, accessibility, and copy.
  You do not need to invoke this — Claude Code follows it
  whenever it is building UI.

SEO SKILL (agents/seo-skill.md)
  Automatically applied to all pages. Covers metadata, Open
  Graph tags, structured data, sitemap, and robots.txt.
  You do not need to invoke this — Claude Code follows it
  whenever it builds a page.

QA SKILL (agents/qa-skill.md)
  Run this when the site feels finished and you want to check
  it before sharing with the client. Say:
      "Run QA on this site"
  Claude Code will work through a full checklist and report
  any issues with severity levels.

IMAGE GENERATION SKILL (agents/image-gen-agent.md)
  Optional. Use this when a project needs AI-generated imagery
  instead of placeholder images. Requires a Gemini API key.
  Say:
      "Use the image gen agent to generate images for this project"
  Follow the setup instructions inside the file.


================================================================
STEP 6 — DEPLOY TO VERCEL
================================================================

Before deploying, make sure your latest changes are pushed to GitHub:
  git add .
  git commit -m "describe what you built"
  git push

FIRST DEPLOY
  1. Go to vercel.com and log in
  2. Click "Add New Project"
  3. Click "Import Git Repository"
  4. Find your repo and click Import
  5. Vercel auto-detects Next.js — no configuration needed
  6. Add your environment variables (same as your .env.local)
  7. Click Deploy

You will get a live URL like your-project.vercel.app within
a couple of minutes. Share this with the client for review.

SUBSEQUENT DEPLOYS
  Every time you push to GitHub, Vercel automatically redeploys.
  You do not need to do anything in Vercel after the first deploy.


================================================================
PUSHING CHANGES TO GITHUB
================================================================

After making changes, save your files in VS Code (Cmd+S) then
run these three commands in your terminal:

  git add .
  git commit -m "brief description of what you changed"
  git push

Always write a clear commit message so your team knows what changed.
Examples:
  "feat: add contact form"
  "fix: mobile nav layout"
  "update: homepage copy"


================================================================
NEED HELP?
================================================================

  Claude Code confused? → Close it, reopen it, try again
  Vercel build failing? → Check the build log in Vercel dashboard
                          and paste the error into Claude Code
  Git issues?           → Paste the error into Claude Code and
                          ask it to help you fix it
