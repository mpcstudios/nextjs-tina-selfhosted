 MPC STUDIOS — NEXT.JS + TINACMS SELF-HOSTED STARTER
=====================================================
This is the MPC Studios starter template for building client websites
with self-hosted TinaCMS, Next.js 15, Tailwind CSS, and Claude Code.

Auth model: TinaCMS commits via the MPC Studios CMS GitHub App.
Secret model: All secrets live in Infisical and sync to Vercel.

Three setup paths — pick the one that fits:

  FASTEST: Deploy button + Claude Code drives the rest (recommended)
  HUMAN:   Same steps, doing each one yourself in UI/CLI
  PLANNED: scripts/spinup-site.sh — single-command future automation


================================================================
WHAT'S IN THIS TEMPLATE
================================================================

CLAUDE.md
  The main instruction file for Claude Code. Contains the New-site
  setup ritual that Claude Code follows when you say "run the
  new-site setup" — this is where most of the post-deploy work
  is documented.

PROJECT_BRIEF.md
  Where you describe the client project. Fill this out completely
  before opening Claude Code to build the site.

.env.example
  Reference for the env vars that will live in Infisical. You don't
  fill this file in by hand for production deploys — it's only
  useful for `dev:prod` testing locally.

tina/
  - tina/config.tsx — main CMS config (auth, media, collections)
  - tina/database.ts — database layer (local LevelDB or Upstash Redis)
  - tina/github-app-provider.ts — custom GitProvider using the
    GitHub App for content commits
  - tina/collections/ — content type definitions (page, post)

pages/api/
  API routes for TinaCMS backend auth and S3 media uploads.
  Uses the Pages Router (required by TinaCMS).

content/
  Markdown and JSON content files. Git-backed — edits from the
  CMS are committed to the repo as mpc-studios-cms[bot].

scripts/
  - patch-admin.sh + admin-branding.html — replaces TinaCMS logos
    with MPC Studios branding (runs as part of build)

patches/
  pnpm patches for ESM/CJS compatibility. Do not delete.

agents/
  Claude Code skills for design, SEO, QA, and image generation.

FASTEST — DEPLOY BUTTON + CLAUDE CODE DRIVES THE REST
================================================================

STEP 1: CLICK THE DEPLOY BUTTON

  Open README.md and click "Deploy with Vercel". Or use this URL:

    https://vercel.com/new/clone?repository-url=https%3A%2F%2Fgithub.com%2Fmpcstudios%2Fnextjs-tina-selfhosted-v2&project-name=my-tina-site&stores=%5B%7B%22type%22%3A%22kv%22%7D%5D

  IMPORTANT: On the Vercel screen, change "Git Scope" to mpcstudios
  so the new repo is created under the org, not your personal GitHub.

  This creates:
    1. A new repo at github.com/mpcstudios/<your-slug>
    2. A Vercel project linked to that repo
    3. An Upstash KV database (auto-sets KV_* env vars in Vercel)

  The first deploy will succeed but /admin won't work yet — secrets
  come next, from Infisical.


STEP 2: CLONE LOCALLY AND OPEN CLAUDE CODE

    git clone git@github.com:mpcstudios/<your-slug>.git
    cd <your-slug>
    pnpm install
    claude

  Once Claude Code is open, say:

    "Run the new-site setup."

  Claude Code reads CLAUDE.md and walks you through:
    - Verifying prerequisites
    - Installing the MPC Studios CMS GitHub App on your repo
      (one browser click — paste the installation ID back to Claude)
    - Creating a new Infisical project (site-<your-slug>)
    - Setting up Secret Imports from mpc-shared
    - Populating per-site secrets silently
    - Configuring Infisical → Vercel sync
    - Triggering a real deploy
    - Verifying /admin saves commit as mpc-studios-cms[bot]

  Total time: ~5 minutes plus one browser click.


STEP 3: BUILD THE SITE

  With the CMS working, fill out PROJECT_BRIEF.md, then tell Claude:

    "Read PROJECT_BRIEF.md and build the homepage."

  Iterate page by page. Every push to GitHub auto-deploys to Vercel.


================================================================
HUMAN — DOING IT MANUALLY
================================================================

If you'd rather drive the post-deploy steps yourself instead of
asking Claude Code, follow the New-site setup ritual section in
CLAUDE.md. Same 10 steps, you run each command and click each UI.

The steps in summary:

  1. Click the deploy button (Step 1 above) to create the cloud resources
  2. gh repo view --json name --jq .name → note your slug
  3. Open https://github.com/apps/mpc-studios-cms/installations/new,
     install on your repo only, copy the installation ID from the URL
  4. Open https://app.infisical.com → New Project → site-<slug>
  5. Run `infisical init` in the cloned repo, pick site-<slug>
  6. In Infisical UI: site-<slug>/prod → Add Secret Import from
     mpc-shared/prod → /
  7. Push per-site secrets via CLI:
       openssl rand -base64 32  → use as NEXTAUTH_SECRET
       infisical secrets set --env=prod --silent \
         GITHUB_APP_INSTALLATION_ID=<id> \
         S3_MEDIA_ROOT=<slug> \
         ENABLE_EXPERIMENTAL_COREPACK=1 \
         NEXTAUTH_SECRET=<generated> > /dev/null 2>&1
  8. Pull KV vars from Vercel (vercel env pull → temp file → push to
     Infisical silently → shred temp)
  9. Set up Infisical → Vercel sync in BOTH site-<slug> and mpc-shared
     projects (Secret Syncs → Vercel → Overwrite + Disable Deletion ON)
  10. git commit --allow-empty -m "Trigger deploy" && git push
  11. /admin login: tinauser / tinarocks → change password → edit content
  12. Verify latest commit author = mpc-studios-cms[bot]

CLAUDE.md has the exact CLI commands. Refer to it.

TINACMS QUICK REFERENCE
================================================================

ADMIN PANEL
  /admin — visual content editor with live preview

LOCAL DEV
  pnpm dev
    Local mode — no auth, edits save to filesystem.

  infisical run --env=prod -- pnpm dev:prod
    Production mode — pulls secrets from Infisical, real Redis,
    real GitHub commits via the App. Use only when debugging
    something that requires the full prod path.

CONTENT FILES
  content/pages/ — page content (markdown)
  content/posts/ — blog posts (markdown)
  content/users/ — CMS user accounts (JSON)

ADDING A NEW COLLECTION
  1. Create tina/collections/your-collection.ts
  2. Import and add to collections array in tina/config.tsx
  3. Create content/your-collection/ directory
  4. Add a sample markdown file
  5. Run pnpm dev to generate types

DEFAULT CMS USER
  Username: tinauser
  Password: tinarocks
  (password change required on first login)


================================================================
NEED HELP?
================================================================

  Claude Code confused?      Close it, reopen, try again
  Vercel build failing?      Check the Vercel build log, paste error
                             into Claude Code
  TinaCMS admin broken?      vercel logs --environment production
  Auth not working?          Verify NEXTAUTH_SECRET and KV env vars
                             synced from Infisical to Vercel
  GitHub App save failing?   Verify GITHUB_APP_ID, GITHUB_APP_PRIVATE_KEY,
                             GITHUB_APP_INSTALLATION_ID are present in
                             Vercel env (synced from Infisical)
  S3 upload failing?         Verify S3_ACCESS_KEY/S3_SECRET_KEY are
                             synced from mpc-shared (service user keys)
